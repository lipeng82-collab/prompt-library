# 政策信息质量打分模型设计

> 为湘小报和policy-tracker-analyst建立自动化的政策重要性评分体系
> 版本：v1.0.0 | 设计日期：2026-05-08

---

## 一、设计目标

### 核心问题
- 每天从8个信源抓取的政策信息可能有30-50条，人工筛选不现实
- 需要自动化判定"哪些政策值得进早报"
- 需要区分"全国性政策"和"湖南相关政策"的不同权重

### 对标参考
- AIHOT的"精选模式"：从所有信息中挑出值得关注的信息
- AIHOT的"质量打分"：模型发布/产品发布/行业动态/论文研究/技巧观点五分类

## 二、评分模型架构

### 2.1 总体公式

```
政策重要性总分 = 基础影响分 × 时效性系数 × 关联度系数 × 湖南加分

其中：
- 基础影响分（1-10分）：政策本身的层级和影响范围
- 时效性系数（0.5-1.5）：政策新鲜度和紧迫性
- 关联度系数（0.5-1.5）：与湖南国资国企的关联程度
- 湖南加分（0-3分）：是否直接涉及湖南/省属国企

最终星级 = round(总分 / 2)  →  1-5星制
```

### 2.2 各维度详细设计

#### 维度1：基础影响分（1-10分）

| 评分项 | 权重 | 评分标准 | 示例 |
|--------|------|---------|------|
| 发布机构层级 | 40% | 国务院(10)/部委(8)/省级政府(7)/省级部门(6)/市级(4) | 国务院发文=10分 |
| 文件类型 | 30% | 法律法规(10)/条例(9)/意见(8)/通知(7)/方案(6)/讲话(4) | 正式文件>领导讲话 |
| 涉及金额 | 15% | ≥100亿(10)/≥10亿(8)/≥1亿(6)/<1亿(4)/无(3) | 百亿级产业政策=10分 |
| 涉及企业数量 | 15% | 全部央企(10)/部分央企(8)/省属国企(7)/单一企业(5) | 涉及全部省属国企=7分 |

**计算示例**：
- 国务院印发《国企改革深化方案》（国务院+意见+无金额+全部央企）
  = 10×0.4 + 8×0.3 + 3×0.15 + 10×0.15 = 4+2.4+0.45+1.5 = **8.35分**

- 湖南省国资委发布《某省属企业考核办法》（省级部门+通知+无金额+单一企业）
  = 6×0.4 + 7×0.3 + 3×0.15 + 5×0.15 = 2.4+2.1+0.45+0.75 = **5.7分**

#### 维度2：时效性系数（0.5-1.5）

| 时效特征 | 系数 | 判定标准 |
|---------|------|---------|
| 当日发布 | 1.5 | 发布日期=今日 |
| 3日内 | 1.3 | 发布日期在3天内 |
| 1周内 | 1.1 | 发布日期在7天内 |
| 1月内 | 1.0 | 发布日期在30天内 |
| 超1月 | 0.8 | 发布日期超过30天 |
| 预告信号 | 1.2 | 征求意见稿/预告/调研（尚未正式发布但信号明确）|

#### 维度3：关联度系数（0.5-1.5）

| 关联特征 | 系数 | 判定标准 |
|---------|------|---------|
| 直接相关 | 1.5 | 标题/正文明确出现"国资监管"/"国企改革"/"省属企业"等关键词 |
| 强相关 | 1.3 | 涉及产业基金/战新产业/资本运作等国资核心领域 |
| 中度相关 | 1.1 | 涉及宏观经济/财税金融/区域发展（对国企有间接影响）|
| 弱相关 | 0.9 | 一般性经济政策（如消费促进、就业政策）|
| 无关 | 0.5 | 与国资国企无明显关联 |

#### 维度4：湖南加分（0-3分）

| 加分项 | 分值 | 判定标准 |
|--------|------|---------|
| 湖南专属 | +3 | 明确涉及湖南省/湖南企业/湖南政策 |
| 中部地区 | +1 | 涉及中部六省（含湖南） |
| 全国普适 | 0 | 全国性政策，不针对特定地区 |

## 三、完整评分示例

### 示例1：高价值条目（⭐⭐⭐⭐⭐）

**政策**：国务院印发《关于深化国资国企改革2026年重点工作安排》

| 维度 | 计算 | 得分 |
|------|------|------|
| 基础影响分 | 国务院(10)×0.4 + 意见(8)×0.3 + 无金额(3)×0.15 + 全部央企(10)×0.15 | 8.35 |
| 时效性系数 | 当日发布 | ×1.5 |
| 关联度系数 | 直接相关（标题含"国资改革"） | ×1.5 |
| 湖南加分 | 全国普适 | +0 |
| **总分** | 8.35 × 1.5 × 1.5 + 0 | **18.8分** |
| **星级** | round(18.8 / 2) | **⭐⭐⭐⭐⭐（上限5星）** |

### 示例2：中价值条目（⭐⭐⭐）

**政策**：湖南省国资委召开省属企业一季度经济运行分析会

| 维度 | 计算 | 得分 |
|------|------|------|
| 基础影响分 | 省级部门(6)×0.4 + 通知(7)×0.3 + 无金额(3)×0.15 + 省属国企(7)×0.15 | 5.7 |
| 时效性系数 | 3日内 | ×1.3 |
| 关联度系数 | 直接相关 | ×1.5 |
| 湖南加分 | 湖南专属 | +3 |
| **总分** | 5.7 × 1.3 × 1.5 + 3 | **14.1分** |
| **星级** | round(14.1 / 2) | **⭐⭐⭐⭐（7分制→4星）** |

### 示例3：低价值条目（⭐⭐）

**政策**：某市发布《关于进一步促进消费增长的若干措施》

| 维度 | 计算 | 得分 |
|------|------|------|
| 基础影响分 | 市级(4)×0.4 + 通知(7)×0.3 + 无金额(3)×0.15 + 无(3)×0.15 | 4.6 |
| 时效性系数 | 当日发布 | ×1.5 |
| 关联度系数 | 弱相关（消费促进） | ×0.9 |
| 湖南加分 | 非湖南 | +0 |
| **总分** | 4.6 × 1.5 × 0.9 + 0 | **6.2分** |
| **星级** | round(6.2 / 2) | **⭐⭐⭐（3星，可进早报但放末尾）** |

## 五、技术实现

### 5.1 Python评分模块

```python
class PolicyScorer:
    """政策信息质量打分器"""
    
    # 关键词库（上文已定义）
    DIRECT_KEYWORDS = [...]
    STRONG_KEYWORDS = [...]
    MODERATE_KEYWORDS = [...]
    HUNAN_KEYWORDS = [...]
    
    # 机构层级映射
    ORG_LEVELS = {
        "国务院": 10, "全国人大": 10,
        "国资委": 8, "发改委": 8, "财政部": 8, "工信部": 8,
        "湖南省政府": 7, "湖南省国资委": 6,
        "长沙市政府": 4, "株洲市政府": 4,
    }
    
    # 文件类型映射
    DOC_TYPES = {
        "法律": 10, "条例": 9, "意见": 8, "通知": 7, 
        "方案": 6, "办法": 6, "讲话": 4, "会议": 4
    }
    
    def score(self, title: str, content: str, org: str, doc_type: str, 
              publish_date: datetime, amount: float = 0) -> dict:
        """对单条政策信息进行评分"""
        
        # 1. 基础影响分
        base_score = self._calc_base_score(org, doc_type, amount)
        
        # 2. 时效性系数
        time_factor = self._calc_time_factor(publish_date)
        
        # 3. 关联度系数
        relevance_factor = self._calc_relevance_factor(title, content)
        
        # 4. 湖南加分
        hunan_bonus = self._calc_hunan_bonus(title, content, org)
        
        # 5. 总分
        total = base_score * time_factor * relevance_factor + hunan_bonus
        
        # 6. 星级
        stars = min(5, max(1, round(total / 3)))
        
        # 7. 分类
        category = self._classify(title, content)
        
        return {
            "base_score": round(base_score, 2),
            "time_factor": time_factor,
            "relevance_factor": relevance_factor,
            "hunan_bonus": hunan_bonus,
            "total_score": round(total, 2),
            "stars": stars,
            "category": category,
            "qualified": total >= 6  # 是否达到早报门槛
        }
```

### 5.2 集成到湘小报工作流

```
Step 1: 采集 → 获取原始政策条目
    ↓
Step 2: 去重 → SimHash去重
    ↓
Step 3: 【评分】→ PolicyScorer.score() 逐条评分
    ↓
Step 4: 筛选 → 仅保留 qualified=True 的条目
    ↓
Step 5: 排序 → 按 total_score 降序排列
    ↓
Step 6: 分类 → 按 category 分组
    ↓
Step 7: 生成 → 按模板生成早报
```

## 六、持续优化机制

### 6.1 人工反馈回路

| 反馈类型 | 来源 | 用途 |
|---------|------|------|
| 遗漏报告 | 用户反馈"某条重要政策没进早报" | 调整关键词库或权重 |
| 噪声报告 | 用户反馈"某条不重要" | 提高该类型门槛 |
| 分类纠错 | 用户反馈"分类错了" | 优化分类规则 |

### 6.2 月度模型校准

```
每月底：
1. 统计本月所有评分分布
2. 对比用户反馈（遗漏/噪声）
3. 调整权重参数（±10%范围内）
4. 更新关键词库
5. 发布新版评分模型
```
