# 湘小报 · 自动政策早报Agent 详细设计

> 对标AIHOT日报模式，打造国资国企领域的自动化政策情报管道
> 版本：v1.0.0 | 设计日期：2026-05-08 | 设计阶段：P0详细设计

---

## 一、定位与目标

### 核心定位
湘小报是湘江研究院的**自动化政策情报雷达**，每天北京时间早8:00自动生成并推送《国资政策早报》，让研究院和合作方在30秒内掌握昨日全国及湖南国资国企领域的关键政策动态。

### 设计目标
| 目标 | 指标 |
|------|------|
| 自动化率 | 从信息采集到早报生成的全流程自动化率≥90% |
| 时效性 | 早报在早8:00前完成生成并推送 |
| 信噪比 | 每日精选政策条目控制在8-15条，每条附一句话影响判断 |
| 覆盖率 | 核心信源（国务院/国资委/湖南省）覆盖率≥95% |

## 五、自动化工作流

### 5.1 定时任务配置

| 任务 | 触发时间 | 动作 |
|------|---------|------|
| 信息采集 | 每日 06:00 | 并行抓取8个信源 |
| 去重与评分 | 每日 06:30 | SimHash去重 + 质量打分 |
| AI生成 | 每日 07:00 | 调用deep-research精简模式生成早报 |
| 质量检查 | 每日 07:30 | 湘小法快速审查（合规检查） |
| 推送分发 | 每日 08:00 | 向订阅列表推送 |
| 日志归档 | 每日 08:30 | 存入湘小档 + 更新政策数据库 |

### 5.2 异常处理

| 异常场景 | 处理策略 |
|---------|---------|
| 某信源抓取失败 | 记录日志，继续处理其他信源，失败信源标记为"待修复" |
| 当日政策条目<5条 | 自动降低重要性阈值（3星→2星），补充行业动态 |
| AI生成超时 | 降级为"简版早报"（仅标题+链接，无摘要） |
| 湘小法审查不通过 | 人工介入标记，延迟推送，记录到异常日志 |

## 六、与现有Agent体系的协作接口

```
湘小报（采集+生成）
    │
    ├──→ 政策条目 → policy-tracker-analyst（深度解读储备）
    │
    ├──→ 高质量条目（⭐⭐⭐⭐⭐） → 湘小选（选题推荐信号）
    │
    ├──→ 湖南专属条目 → 湘小研（优先进入RAG知识库）
    │
    ├──→ 预告信号 → 湘小策（战略预判素材）
    │
    ├──→ 完整早报 → 湘小档（归档+历史检索）
    │
    └──→ 行动建议 → 大总管（每日任务优先级参考）
```

## 七、技术实现路径

### Phase 1（2周内）：MVP
- [ ] 确定8个核心信源的RSS/可抓取URL
- [ ] 实现基础WebFetch采集脚本（Python）
- [ ] 实现SimHash去重模块
- [ ] 实现Markdown早报模板生成
- [ ] 手动触发试运行（连续7天）

### Phase 2（1个月内）：自动化
- [ ] 接入WorkBuddy automation系统（每日06:00自动触发）
- [ ] 接入质量打分模型（与policy-tracker-analyst协作）
- [ ] 接入湘小法快速审查（合规检查）
- [ ] 接入企业微信/邮件推送

### Phase 3（2个月内）：智能化
- [ ] 接入预告信号捕获（征求意见稿/调研动态）
- [ ] 实现"周末合订版"和"回溯查询"
- [ ] 接入外部RSS订阅和API输出
- [ ] 用户反馈回流（打开率/点击率追踪）
