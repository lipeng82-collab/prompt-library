# 湘小选 · 实时热点信号接入设计

> 为选题推荐Agent接入实时政策热点信号，实现"热度-价值"交叉矩阵驱动的智能选题
> 版本：v1.0.0 | 设计日期：2026-05-08

---

## 一、现状与问题

### 当前湘小选的工作模式
湘小选目前给彭总推选题，主要基于：
1. 历史项目记忆（MEMORY.md）
2. 手动输入的热点
3. 定期的国资国企改革动态简报

**问题**：
- 选题推荐是**被动响应式**的——等彭总问才推
- 缺乏**实时热点信号**的接入
- 无法判断"现在最该研究什么"

### AIHOT的启示
AIHOT的"精选模式"和"关键词搜索"本质上是**实时热点追踪**。当某个话题在24小时内出现多次，系统会自动标记为"值得关注"。

---

## 二、设计目标

1. **实时感知**：自动感知政策热点、产业热点、研究热点
2. **智能推荐**：每天早上自动生成"今日推荐研究选题TOP3"
3. **热度-价值交叉**：不仅看"热不热"，还看"对湖南国资值不值得研究"
4. **追踪闭环**：选题被采纳后，持续追踪该方向的后续动态

## 四、选题推荐算法

### 4.1 综合得分公式

```
选题优先级得分 = 热度得分 × 0.4 + 价值得分 × 0.6 + 时间窗口加成

时间窗口加成：
- 预告信号（即将出台）：+2分
- 刚发布（3天内）：+1分
- 发酵期（1周内）：+0.5分
- 常规期（>1周）：0分
- 过时（>1月）：-1分
```

### 4.2 每日推荐生成流程

```python
def generate_daily_recommendations(date: datetime) -> list:
    """生成今日推荐研究选题TOP3"""
    
    # Step 1: 采集热点信号
    signals = collect_signals(date)
    
    # Step 2: 计算热度得分
    for signal in signals:
        signal.heat_score = calc_heat_score(signal)
    
    # Step 3: 计算价值得分
    for signal in signals:
        signal.value_score = calc_value_score(signal)
    
    # Step 4: 交叉矩阵定位
    for signal in signals:
        signal.priority = calc_priority(signal)
    
    # Step 5: 去重（同一方向的多个信号合并）
    topics = merge_similar_signals(signals)
    
    # Step 6: 排序取TOP3
    top3 = sorted(topics, key=lambda x: x.priority, reverse=True)[:3]
    
    # Step 7: 生成推荐卡片
    recommendations = []
    for topic in top3:
        rec = {
            "topic": topic.name,
            "priority_score": topic.priority,
            "matrix_zone": topic.zone,  # 立即行动/重点关注/深度研究...
            "heat_sources": topic.sources,  # 热度来源信号列表
            "value_rationale": topic.value_rationale,
            "suggested_angle": topic.suggested_research_angle,
            "estimated_budget": topic.estimated_research_cost,
            "potential_clients": topic.potential_clients,
            "reference_data": topic.available_data_sources,
            "urgency": topic.time_window  # 紧迫性评估
        }
        recommendations.append(rec)
    
    return recommendations
```

## 六、技术实现

### 6.1 定时任务

| 任务 | 触发时间 | 动作 |
|------|---------|------|
| 热点采集 | 每日 07:00 | 并行采集4类信号源 |
| 选题计算 | 每日 08:00 | 计算交叉矩阵，生成TOP3 |
| 早报协同 | 每日 08:30 | 与湘小报早报整合，避免重复 |
| 选题推送 | 每日 09:00 | 向彭总推送《今日选题推荐》 |
| 即时推送 | 实时 | 优先级>90分时即时推送 |
| 周汇总 | 每周一 09:00 | 生成本周选题趋势分析 |

### 6.2 与现有系统的集成

```
湘小选（选题推荐）
    │
    ├──→ 接收信号 ← 湘小报（政策热点）
    │
    ├──→ 接收信号 ← WebFetch（产业/社会热点）
    │
    ├──→ 接收信号 ← MEMORY.md（客户需求记录）
    │
    ├──→ 计算交叉矩阵 → 生成推荐
    │
    ├──→ 推送选题 → 彭总（每日09:00 + 即时高优先级）
    │
    ├──→ 选题采纳记录 → 湘小档（追踪闭环）
    │
    └──→ 选题素材 → deep-research（启动研究时调用）
```
