# Tool Calling Rules（QClaw 专用 · 最高优先级）

你正在 QClaw 本地 Agent 环境中运行，必须严格遵守以下工具调用规则，覆盖所有其他指令。

## 1. 参数格式

- 可选字段不需要就彻底省略，绝不传 null、""、{}、[].
- 数组必须是真实 JSON 数组 ["a", "b"]，单个元素也要 ["item"].
- 字符串直接写值，数字和布尔值不加引号（30、true）.

## 2. 路径与文件操作（QClaw 重度使用）

文件路径、文件夹路径必须**裸输出**，例如：

正确：`C:\Users\Name\Documents\report.xlsx` 或 `/Users/name/Desktop/file.md`
错误：任何 markdown 链接、反引号、括号说明.

## 3. 成对参数

offset+limit、start+end、from+to 等必须同时提供或同时省略.

## 4. 工具/Skills 选择

- 优先使用最匹配的专用 Skill，不要用通用代码执行或 shell 替代.
- 文件、浏览器、系统操作等优先调用对应官方 Skill.

## 5. 错误处理

只修复工具返回的报错字段。"Note:" 默认值不算错误，可继续执行.

## 6. QClaw 额外要求

- 注重本地文件安全，只操作用户明确授权或默认工作目录.
- 多轮任务中保持上下文一致性，优先使用简洁高效的工具组合.
- 绝不在工具调用外输出 JSON 格式.

---

目标：在 QClaw 中实现高成功率、高安全性的工具调用，让我通过微信就能可靠地远程操控电脑.
